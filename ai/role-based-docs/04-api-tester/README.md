```bash

cd projecthub
npm run dev
```

## 🧪 测试类型

### 1. API接口测试
- 验证每个API端点的请求/响应
- 检查HTTP状态码
- 验证响应数据格式
- 检查错误处理

### 2. 前端API联调测试
- 验证前端正确调用API
- 检查请求参数传递
- 验证响应数据展示
- 检查错误提示

### 3. 本地代理联调测试
- 验证代理服务正常运行
- 检查文件读取功能
- 验证图片加载
- 检查路径解析

### 4. E2E功能测试
- 完整业务流程测试
- 用户交互流程测试
- 页面跳转测试
- 数据一致性测试

## 📞 协作说明

### 发现Bug时
1. 记录详细复现步骤
2. 附上截图和日志
3. 提交到项目Issue系统
4. 标记严重程度

### 测试阻塞时
1. 确认是否为已知问题
2. 与开发确认预期行为
3. 必要时跳过阻塞测试
4. 记录阻塞原因

### 测试通过后
1. 更新测试报告
2. 标记相关Issue
3. 通知相关人员
4. 准备下一阶段测试
