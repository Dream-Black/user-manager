## 🎨 设计原则

### 1. 简洁高效
- 信息层级清晰
- 减少视觉噪音
- 突出内容本身

### 2. 专注阅读
- 漫画内容最大化为首要目标
- 导航和工具最小化
- 沉浸式阅读体验

### 3. 一致性
- 与现有TDesign设计语言保持一致
- 复用项目已有设计模式
- 统一交互反馈

## 📐 设计尺寸

### 设备适配
| 设备 | 宽度范围 | 基准 |
|------|----------|------|
| 手机 | 320px - 767px | 375px |
| 平板 | 768px - 1023px | 768px |
| 桌面 | 1024px - 1439px | 1280px |
| 大屏 | ≥ 1440px | 1440px |

### 间距系统
基于8px网格系统：
```
4px  / 8px / 12px / 16px / 24px / 32px / 48px / 64px
```

### 栅格系统
| 设备 | 列数 | 间距 | 边距 |
|------|------|------|------|
| 手机 | 2列 | 12px | 16px |
| 平板 | 4列 | 16px | 24px |
| 桌面 | 12列 | 24px | 32px |

## 🔧 与开发协作

### 设计稿要求
1. **标注尺寸**：间距、字号、颜色值
2. **标注状态**：默认、悬停、点击、禁用
3. **标注交互**：点击区域、过渡效果
4. **标注动效**：动画时长、缓动函数

### 交付流程
```
设计稿 → 开发评审 → 修改确认 → 交付开发
   ↑                         ↓
   └───── 开发反馈 ← ← ← ← ← ┘
```

### 沟通要点
- 开发对设计有疑问及时提出
- 设计调整需同步更新文档
- 涉及交互改动需提前沟通
