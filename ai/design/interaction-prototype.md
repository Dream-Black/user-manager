# ProjectHub 交互原型设计

**版本**: v1.0
**更新日期**: 2026-04-13
**设计理念**: 大厂级体验、极致交互、自然动效

---

## 一、设计原则

### 1.1 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   "每一次交互都应该让用户感到愉悦"                            │
│                                                             │
│   • 自然：动效符合物理规律                                   │
│   • 即时：响应时间 < 100ms                                  │
│   • 清晰：状态变化一目了然                                  │
│   • 愉悦：微妙的反馈让人心情舒畅                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 交互黄金法则

| 法则 | 描述 | 示例 |
|------|------|------|
| **及时反馈** | 每个操作都有视觉/触觉反馈 | 按钮点击缩小至95% |
| **状态可见** | 当前位置和状态清晰可见 | 侧边栏高亮当前项 |
| **防错设计** | 防止误操作，提供确认 | 删除前弹窗确认 |
| **识别替代记忆** | 减少用户记忆负担 | 显示完整路径 |
| **灵活性** | 同时支持新手和专家 | 快捷键+点击 |
| **美学一致性** | 视觉风格统一 | 间距、圆角统一 |

## 三、交互动效规范

### 3.1 通用动效

| 动效类型 | 参数 | 适用场景 |
|----------|------|----------|
| **淡入** | opacity 0→1, 200ms ease-out | 元素出现 |
| **淡出** | opacity 1→0, 150ms ease-in | 元素消失 |
| **上浮** | translateY(8px)→0, 200ms ease-out | 悬停、出现 |
| **下浮** | translateY(0)→8px, 150ms ease-in | 消失 |
| **缩放** | scale(0.95)→1, 200ms ease-out | 弹窗打开 |
| **按压** | scale(1)→0.95, 50ms | 按钮点击 |
| **弹跳** | scale(1)→1.05→1, 300ms ease-out | 成功反馈 |

### 3.2 状态过渡

```
颜色变化：200ms ease-in-out
阴影变化：200ms ease-out
边框变化：150ms ease-in-out
transform：200ms ease-out
```

### 3.3 延迟规范

```
列表项依次出现：stagger 50-100ms
模态框内容依次出现：stagger 30-50ms
时间线节点依次出现：stagger 100-150ms
```

### 3.4 性能要求

```
动画帧率：60fps
主线程阻塞：< 16ms
动画期间不触发重排
优先使用 transform 和 opacity
```
